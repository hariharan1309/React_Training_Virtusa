                                                                                                                                                                Jools Shivesh Chandra 7 years ago

                                                                                                                                                                Be very cautious when using alerts with preventDefault . An alert will often stop the default behaviour from being prevented. This is because it will pause the thread and for security/usability reasons some browsers will continue with their default behaviour. Obviously you wouldn't have the alert in your release code, but even for debugging and exploratory code it can completely throw you off when you're trying to determine what you can and cannot prevent. I'd say that alerts suffer far too much from the observer effect and you're better sticking to console.logs.

                                                                                                                                                                Also, I might be wrong about this, but I think you should be using event.which rather than keyCode as "which" is the field that jQuery's documentation states is normalized across browsers.
